# 重复工作流候选清单

| 优先级 | 工作流 | 证据 | 适合沉淀为 | 触发条件 |
| --- | --- | --- | --- | --- |
| P0 | 业务 API 变更 | `app/api/**/route.ts`、`lib/validators.ts`、`lib/response.ts` | AGENTS 规则 + 项目 skill | 新增/修改 API、导出、状态变更 |
| P0 | 多管家派单状态流转 | `prisma/schema.prisma`、`lib/order-status.ts`、`app/api/orders/**` | validator + ReviewWorker | 触及派单、取消、确认、完成、拒单 |
| P0 | Web 与小程序双端联动 | `app/(admin)/**`、`miniprogram/pages/**`、`miniprogram/services/**` | 项目 skill + smoke checklist | 同一业务在后台和小程序都有入口 |
| P1 | 账号、微信绑定、权限 | `lib/auth.ts`、`lib/permissions.ts`、`app/api/miniprogram/auth/**` | SecurityReviewWorker | 登录、绑定、解绑、账号启停、密码重置 |
| P1 | 财务与导出 | `lib/finance.ts`、`lib/export.ts`、`app/api/export/**` | validator | 结算、金额、Excel 导出 |
| P1 | 操作日志与通知 | `lib/logger.ts`、`lib/notification.ts`、`lib/notifications-center.ts` | AGENTS 规则 | 关键业务动作新增或状态变化 |
| P1 | README/回执/交接 | `README.md`、本轮 PDF | 通用产物质量 validator | 用户要求总结、交接、审计、报告 |
| P2 | 数据清理脚本 | `scripts/clean-*.ts` | runner 候选 | 批量修改历史数据 |
| P2 | 备份发布脚本 | `scripts/backup-to-github.sh` | 高风险确认规则 | 涉及 git push、远端备份、发布 |

## 失败模式候选

| 问题 | 证据路径 | 频率判断 | 严重程度 | 根因 | 建议 |
| --- | --- | --- | --- | --- | --- |
| 缺少项目级 agent 入口 | 未发现项目 `AGENTS.md` | 高 | 高 | 规则散落在 README 和代码 | 生成并人工安装 AGENTS |
| 完成定义偏弱 | `README.md` 功能清单多，验证命令少 | 中 | 中 | 报告偏状态罗列 | 引入 validator 和回执格式 |
| 双端联动风险 | `app/**` 与 `miniprogram/**` 并存 | 高 | 高 | Web/小程序 API 契约需要同步 | 引入项目 skill |
| 业务状态回归风险 | `lib/order-status.ts`、多条 assignment 状态 | 高 | 高 | 多管家订单边界多 | 引入 ReviewWorker + fixture |
| 高风险脚本误执行 | `scripts/backup-to-github.sh` | 低 | 高 | 推送/备份会影响外部状态 | AGENTS 中强制人工确认 |
